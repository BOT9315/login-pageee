01

What Was Built

Before writing a single line, we defined what the page needed to do and feel like.

🌫️

Glassmorphism UI

Frosted-glass inputs and buttons layered over a full-bleed background image using rgba() and backdrop-filter.

🔄

Slide Transitions

Login and Register panels swap via CSS left/right positioning + opacity, triggered by vanilla JS — no libraries.

📱

Responsive Layout

Two breakpoints: hamburger nav at 786px, full-width stacked form at 540px. Pure CSS media queries.

🔡

CDN-loaded Assets

Poppins font via Google Fonts and Boxicons via unpkg — zero local assets except the background image.

⚡

Zero Build Tools

Open index.html in any browser and it works. No npm, no webpack, no server needed.

🎯

Clean Separation

HTML handles structure, CSS handles all styling and animation, JS handles only toggling — textbook separation of concerns.

Step-by-Step Build

Exactly how every part of the page was constructed, in order.

01

Set up HTML boilerplate & load external assets

Created index.html with the standard HTML5 doctype, then linked Boxicons and Poppins via CDN in the <head>. No npm installs — just two link tags.

HTML
<link href='https://unpkg.com/boxicons@2.1.4/css/boxicons.min.css' rel='stylesheet'> <link href='https://fonts.googleapis.com/css2?family=Poppins:wght@400;500;600;700' rel='stylesheet'> <link rel="stylesheet" href="style.css">
02

Build the navbar with responsive menu

The nav uses flexbox with space-around to spread logo, links, and buttons. A hidden hamburger icon activates on mobile screens via a JS class toggle.

CSS
.nav { position: fixed; display: flex; justify-content: space-around; width: 100%; height: 100px; background: linear-gradient(rgba(39,39,39,0.6), transparent); /* Fades from dark to transparent for depth */ }
03

Create the dual-panel form box

Both forms (.login-container, .register-container) sit inside a shared .form-box with overflow: hidden. The login panel starts at left: 4px (visible), the register panel starts at right: -520px (hidden off-screen).

CSS
.form-box { position: relative; width: 512px; height: 420px; overflow: hidden; /* clips the hidden panel */ } .login-container { left: 4px; } /* visible */ .register-container { right: -520px; } /* hidden */ .login-container, .register-container { transition: .5s ease-in-out; }
04

Wire up form switching with vanilla JS

Two tiny functions handle the swap. login() slides the login panel in and register panel out. register() does the reverse. Button classes are also toggled for the active white-btn highlight.

JavaScript
function login() { x.style.left = "4px"; y.style.right = "-520px"; x.style.opacity = 1; y.style.opacity = 0; a.className += " white-btn"; // highlight Sign In btn b.className = "btn"; } function register() { x.style.left = "-510px"; y.style.right = "5px"; x.style.opacity = 0; y.style.opacity = 1; b.className += " white-btn"; // highlight Sign Up btn a.className = "btn"; }
05

Apply glassmorphism styling to inputs & buttons

All inputs use rgba white backgrounds with no border, giving that frosted-glass feel. The body background image ties everything together. The wrapper overlay (rgba dark) ensures the forms stay readable.

CSS
body { background: url("images/image.jpeg"); background-size: cover; } .wrapper { background: rgba(39, 39, 39, 0.4); /* dark overlay */ } .input-field { background: rgba(255, 255, 255, 0.2); border: none; color: #fff; border-radius: 30px; } .submit { background: rgba(255, 255, 255, 0.7); }
